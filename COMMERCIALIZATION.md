# 🎯 商業化 Roadmap — 戰鬥/玩法翻新 → 免費上架 App Store

> 目標定義（依 2026-06 決策）：
> **把戰鬥機制、平衡、玩法翻新到「商業品質」，最終以「免費 / 作品集」形式上架 iPhone App Store。**
>
> 「商業品質」= 手感、深度、打磨、完整度達到付費遊戲水準；
> 「免費上架」= **不寫內購（IAP）、不接廣告、不收集資料**，作為作品集 / 履歷作品發佈。
>
> 開發節奏沿用既有約定：**分波次（Wave）推進，每波獨立可上線、可在 iPhone 驗收**。

## Part A — 戰鬥 / 玩法翻新（**第一優先**）

> 原則：先「不加新機制、純提升透明度與手感」（低風險高回報），
> 再逐步加「深度機制」。每個 G 區塊可獨立成一波。

### 🟢 G1. 戰鬥透明度與手感（低風險，最高 CP 值，建議最先做）

不新增機制，只讓「看得懂、打得爽」：

1. **修正爆擊倍率不一致**：`DamageCalculator.ts` 程式用 `CRIT_MULTIPLIER = 2.0`，但介面註解寫 ×1.5。先拍板數值（建議 ×1.5 + 視覺強調，避免運氣決定戰局，呼應 DESIGN.md「高命中低變異」），統一程式與註解。
2. **戰前傷害預覽面板（FE 風 forecast）**：選定攻擊目標時，顯示雙方 HP 條、預估傷害、命中%、爆擊%、**反擊傷害**、以及「這刀會不會擊殺」。目前 tooltip 已有部分資訊，升級成攻守對照雙欄。
3. **威脅範圍疊圖（threat range）**：一鍵切換「敵方可移動+可攻擊範圍」紅色疊圖。這是戰術 SRPG 商業標配，對戰術清晰度提升巨大。
4. **行動前可反悔**：移動後、確認動作前能取消退回移動前位置（quality-of-life）。
5. **打擊回饋打磨**：浮動傷害數字、爆擊放大/震動、MISS 明確提示、擊殺特效。部分已有，統一強化。

### 🟢 G2. 平衡性「框架化」（中風險，發揮你的 Python/後端強項）

把目前散落、靠手感調的數值，變成**可測量、可回歸**的資料：

1. **集中可調參數**：把剋制檔位、DEF 曲線、爆擊倍率、EXP 曲線（目前 50/級、+1/傷害、+5/擊殺、cap lv20）、各兵種基礎數值、技能倍率，集中到單一 `src/data/balance.ts` 常數模組，平衡只改一個檔。
2. **無頭平衡模擬器（balance harness）**：寫一支 Node 腳本，模擬「每兵種 vs 每兵種、同級同裝」N 場對戰，輸出勝率 / 平均擊殺回合數 → 用**數據**而非肉眼判斷平衡。（本質是寫 sim/test，正中你後端背景。）
3. **訂定平衡目標指標**：例如「中性條件下無任何對位勝率 > 65%」「單場 30–50 回合」「每兵種至少 1 強剋 1 被剋」，當作回歸基準。

### 🟡 G3. 深度機制 #1 — 速度 / 追擊（中高風險，深度槓桿最大）

新增 `SPD` 速度屬性：攻方 SPD ≥ 守方 SPD + 門檻 → **二次攻擊（double）**。
火紋招牌機制，讓「重攻單發」與「輕攻連擊」分流，戰術質感大增。
影響：baseStats、Leveling、（可選）裝備重量、forecast 顯示「×2」。

### 🟡 G4. 深度機制 #2 — 方位 / 背擊（中高風險）

追蹤單位面向，從側面 / 背面攻擊 → +命中 / +爆擊 / +傷害。
六角格天生 6 方向，很適合包夾、繞後、夾擊戰術（FFT / Triangle Strategy 核心）。

### 🟡 G5. 深度機制 #3 — 狀態效果框架（中高風險）

把現有的 `stance`（防禦姿態）泛化成 **StatusEffect 系統**：中毒(DoT)、暈眩(跳過行動)、攻/防 up/down、沉默(不能放技)，帶持續回合 + 圖示。
讓技能 / 裝備能掛載豐富效果，是後續所有「花式技能」的地基。

### 🟡 G6. 深度機制 #4 — 羈絆 / 支援系統（中風險，DESIGN.md 既定 TODO）

相鄰我方武將互給 buff（命中/迴避/傷害），並解鎖**配對支援對話**（綁進劇情）。
火紋風，增加重玩價值與角色情感連結（呼應 DESIGN.md 情感目標「留下一段故事」）。

### 🟡 G7. 敵方 AI 升級（中高風險，最影響「商業感」）

目前已有「期望傷害評分」。商業級 AI 再加：集火（湊傷害確保擊殺）、殘血撤退 / 遠程風箏、保護治癒兵 / BOSS、善用地形、避免過度突進、角色化行為（攻擊型/防禦型/支援型）、難度連動智商。

### 🟡 G8. 難度與無障礙（中風險）

- **難度分層**：休閒 / 普通 / 困難（目前 `enemyAttackMul` 0.7/1.0/1.3 已是雛形），擴成同時影響敵方數值 + AI 智商 + 玩家補正。
- **無障礙**：文字放大（已做）、色盲友善的剋制標示、動畫速度（已有）、可常駐顯示威脅範圍。

> **建議起手**：G1 → G2 兩波先做（低風險、立刻有感、且 G2 讓後續所有平衡都可量化驗證），之後依你意願從 G3–G8 挑。

## Part C — 製作品質打磨

1. **存檔可靠性**（重要）：localStorage 在 WKWebView 可能被 iOS 在儲存壓力下清除 → 存檔遺失 → 負評。打包後改用原生儲存（Capacitor Preferences / Filesystem），並先加「匯出 / 匯入存檔」保險；雲端 iCloud 列為後續可選。
2. **新手教學 / onboarding**：第一場戰鬥引導移動、剋制、攻擊預覽、結束回合。商業遊戲必備，降低初見挫折。
3. **裝置實測 + safe-area**：瀏海 / 動態島避讓（CSS `env(safe-area-inset-*)`）、橫屏鎖定、舊機型效能。
4. 版本可見性（已有 git sha 顯示，保留）。

## Part D — iOS 打包（免費，**無 IAP**）

> 你已選「先免費上架（作品集）」→ 不需 StoreKit、不需帳號系統，打包大幅簡化。

**打包路線（建議：Capacitor）**

| 路線 | 說明 |
|---|---|
| **Capacitor（推薦）** | web→iOS 業界標準，把 `dist/` 包進 WKWebView，文件最完整、社群最大；雖免費版用不到內購，但日後若想加付費功能擴充最平滑。桌面版**繼續用現有 Tauri**。 |
| Tauri 2 iOS（沿用現有）| 延用既有 Tauri 殼擴到 iOS，桌面+手機一套工具；行動版較新，但免費 App 無內購需求時也夠用。 |

> 兩者在 iOS 上都是跑 WKWebView，遊戲效能相近。免費路線兩者皆可，**Capacitor 文件/範例最多、對非前端背景最友善**，故列為建議；最終打包路線等 Part A/B 接近完成再定案即可（這步不急）。

**打包檢查清單**

- [ ] 真實 Bundle ID（非 `com.web-srpg.demo`，例如 `com.<你的代號>.westsrpg`）
- [ ] App icon 全尺寸 + 啟動畫面 / splash
- [ ] **所有資產本地內嵌、可離線執行**（已是相對路徑 `./`，幾乎就緒）
   ⚠️ App Store Guideline 4.2「最低功能」：純把網站包成殼會被拒；**本地內嵌、離線可玩的完整遊戲**不在此列，但務必確保離線啟動、有原生 icon/splash。
- [ ] TestFlight 內測 → 正式送審

## Part E — 上架素材與合規（免費 App）

1. **App Store Connect**：建立 App 紀錄、截圖（6.7"/6.5" 等）、描述、關鍵字、支援網址。
2. **隱私標籤**：勾「**不收集資料**」（屬實，因零網路）→ 最容易過。但**仍需提供隱私政策 URL**（可放一頁簡單頁面到 GitHub Pages）。
3. **年齡分級**：奇幻暴力、黑暗主題 → 問卷後約落在 9+/12+。
4. **出口合規**：未用非標準加密 → 通常可宣告豁免。
5. **內容授權（最大法律風險，務必先確認）**：
   - **AI 美術**（Nano Banana / Gemini）：確認 Google 條款允許「商業 / 公開發佈」用途。
   - **BGM / SFX**：確認來源授權（若 Suno/Udio 生成，商用權依方案而定；若購買素材，保留授權證明）。
   - App Store 目前不禁 AI 內容，但**你必須擁有發佈權利、不得侵權**。免費發佈同樣需要。

## 風險清單（Risk Register）

| 風險 | 影響 | 緩解 |
|---|---|---|
| 音樂 / 美術授權不明 | 上架被下架 / 法律 | **最優先**確認所有素材商用權利，留存證明 |
| WKWebView 存檔被清 | 玩家進度遺失、負評 | 改原生儲存 + 匯出/匯入 + 日後 iCloud |
| Guideline 4.2 純殼被拒 | 送審失敗 | 本地內嵌、離線可玩、原生 icon/splash |
| Phaser 在舊 iPhone 效能 | 卡頓 | 盡早裝置實測，必要時降特效 |
| 戰鬥翻新範圍蔓延 | 永遠做不完 | 嚴守分波次、每波獨立可上線 |

## 建議波次順序（小 → 大 / 低風險 → 高風險）

> 對應你「依序全部進行」的推進方式；每波獨立可在 iPhone 驗收。

1. **Wave 1（G1）戰鬥透明度與手感** — 爆擊倍率統一 + 戰前預覽 + 威脅範圍 + 反悔 + 打擊回饋
2. **Wave 2（G2）平衡框架化** — `balance.ts` 集中參數 + 無頭模擬器 + 目標指標
3. **Wave 3（G3）速度 / 追擊** — 二次攻擊系統
4. **Wave 4（G4）方位 / 背擊** — 包夾戰術
5. **Wave 5（G5）狀態效果框架** — 中毒/暈眩/buff/debuff
6. **Wave 6（G6）羈絆 / 支援** — 相鄰 buff + 支援對話
7. **Wave 7（G7）AI 升級** — 集火/撤退/護衛/角色化
8. **Wave 8（G8）難度 + 無障礙**
9. **Wave 9（Part B）內容完整化 + 全英化**
10. **Wave 10（Part C）製作打磨**（存檔/教學/safe-area）
11. **Wave 11（Part D/E）iOS 打包 + 上架素材 + 合規**

> Part D/E 刻意排最後：先把遊戲做到值得上架，打包只是最後一哩。
> 但 **Part E 的「素材授權確認」建議現在就先查**，因為它可能反過來影響美術/音樂要不要重做。
