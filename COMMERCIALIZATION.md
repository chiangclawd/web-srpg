# 🎯 商業化 Roadmap — 戰鬥/玩法翻新 → 免費上架 App Store

> 目標定義（依 2026-06 決策）：
> **把戰鬥機制、平衡、玩法翻新到「商業品質」，最終以「免費 / 作品集」形式上架 iPhone App Store。**
>
> 「商業品質」= 手感、深度、打磨、完整度達到付費遊戲水準；
> 「免費上架」= **不寫內購（IAP）、不接廣告、不收集資料**，作為作品集 / 履歷作品發佈。
>
> 開發節奏沿用既有約定：**分波次（Wave）推進，每波獨立可上線、可在 iPhone 驗收**。

## Part A — 戰鬥 / 玩法翻新（**第一優先**）

> 原則：先「不加新機制、純提升透明度與手感」（低風險高回報），
> 再逐步加「深度機制」。每個 G 區塊可獨立成一波。

### 🟢 G1. 戰鬥透明度與手感（低風險，最高 CP 值，建議最先做）

不新增機制，只讓「看得懂、打得爽」：

1. **修正爆擊倍率不一致**：`DamageCalculator.ts` 程式用 `CRIT_MULTIPLIER = 2.0`，但介面註解寫 ×1.5。先拍板數值（建議 ×1.5 + 視覺強調，避免運氣決定戰局，呼應 DESIGN.md「高命中低變異」），統一程式與註解。
2. **戰前傷害預覽面板（FE 風 forecast）**：選定攻擊目標時，顯示雙方 HP 條、預估傷害、命中%、爆擊%、**反擊傷害**、以及「這刀會不會擊殺」。目前 tooltip 已有部分資訊，升級成攻守對照雙欄。
3. **威脅範圍疊圖（threat range）**：一鍵切換「敵方可移動+可攻擊範圍」紅色疊圖。這是戰術 SRPG 商業標配，對戰術清晰度提升巨大。
4. **行動前可反悔**：移動後、確認動作前能取消退回移動前位置（quality-of-life）。
5. **打擊回饋打磨**：浮動傷害數字、爆擊放大/震動、MISS 明確提示、擊殺特效。部分已有，統一強化。

### ✅ G2. 平衡性「框架化」（Wave 2 已完成）

把散落、靠手感調的數值，變成**可測量、可回歸**的資料：

1. ✅ **集中可調參數**：剋制矩陣、DEF 曲線、爆擊倍率、EXP 曲線、等級差縮放都集中到 `src/data/balance.ts`，是平衡唯一來源（CounterSystem/DamageCalculator/Leveling 改成 import 它）。
2. ✅ **無頭平衡模擬器**：`scripts/balance-sim.ts`（`npm run balance`）直接 import 真實傷害公式，跑每兵種 vs 每兵種各 4000 場 1v1，輸出勝率矩陣 + 整體公平度 + 一邊倒對位。
3. ✅ **平衡目標指標**：`BALANCE_TARGETS`（每兵種對全體平均勝率落在 40–60%；單一對位 ≥90% 標記人工檢視）。

**首次基線診斷（待後續平衡波處理）：**
- ⚠ **cavalry 平均勝率 73%（偏強）**：騎兵近戰、模擬器公平評估，僅被 lance 剋；高基礎數值 + move 5 讓它在對打中壓制全體。**真實過強候選**。
- ⚠ **flier 31% / archer 偏低**：模擬器**刻意不模擬風箏/移動**，飛兵的本體是 move 6 + 無視地形，弓兵是 range 2 拉開距離——故 1v1 數字偏低多屬「設計使然」，非真正過弱。
- 👉 後續開「平衡微調波」時，先壓 cavalry 基礎數值（attack 9→8 或 move 5→4），重跑 `npm run balance` 對照即可。

### ✅ G3. 深度機制 #1 — 速度 / 追擊（Wave 3 已完成）

新增 `speed` 速度屬性（每兵種固定）：攻方 speed − 守方 speed ≥ `DOUBLE_ATTACK_THRESHOLD`(4) → **追擊（二次攻擊）**。攻守各自獨立判定（火紋風）。

- **速度配置**：flier 10 / sword 9 / archer 8 / cavalry 7 / lance 6 / mage 4。
- **戰鬥流程重構**：`executeAttack` 抽出 `performSwing` 共用函式，交戰＝攻方第一擊 → 守方反擊 → 攻方追擊 → 守方追擊；empower 只吃第一擊。
- **forecast 顯示「×2追擊」**：預判面板反映加倍傷害與致命判定。
- **模擬器同步**：`balance-sim.ts` 加入 speed/doubling，驗收後 lance 5→6 避免被全體追擊壓垮。
- **結果**：快速兵（劍/弓/飛）懲罰遲緩法師、lance 變耐打中速坦、cavalry 爆發但無追擊、flier 快速騷擾——對位質感大增。殘留 cavalry 62%（角色使然）、mage 35%（脆皮 + 模擬器未計 range 2 風箏）屬可接受。

### ✅ G4. 深度機制 #2 — 方位 / 背擊（Wave 4 已完成）

追蹤單位面向，從側 / 背攻擊 → 傷害加成（側 ×1.1、背 ×1.25），鼓勵包夾繞後。

- **面向**：以 pixel 空間角度儲存（避開 odd-r 奇偶列鄰格方位問題）。玩家預設面右、敵方面左；**只在移動 / 攻擊時轉向，被打不轉**（故繞背的加成能維持到對方行動）。
- **前點指示**：每單位有一個亮色「前點」標出正面方向（移動/攻擊即更新位置）。
- **判定**：守方→攻方角度與守方面向夾角 ≤60° 正面（無加成）、≥120° 背擊、其間側擊（`FLANK` in balance.ts）。
- **透明度**：forecast 顯示「側擊/背擊」、浮動傷害與 log 標示；攻方攻擊後轉向目標，故守方反擊一律正面（不疊加成）。
- 玩家與 AI 攻擊共用 `executeAttack`/`performSwing`，加成雙向生效。
- 註：面向未進戰中存檔（讀檔回預設面向，影響微小）；AI 尚未主動找背擊（屬 Wave 7 AI）。

### ✅ G5. 深度機制 #3 — 狀態效果框架（Wave 5 已完成）

通用 **StatusEffect 系統**（`types.ts` StatusEffectType / StatusEffect）：

- **效果類型**：poison（每回合損血 DoT）、stun（該回合跳過行動）、atk_up/atk_down、def_up/def_down、silence（不能用主動特技），皆帶 `turnsLeft` + `magnitude` + 圖示 label。
- **Unit 整合**：`statuses[]` + `addStatus`/`tickStatuses`/`hasStatus`/`isStunned`/`isSilenced`；攻/防增減**直接折進 `attack`/`defense` getter**（computeDamage 自動吃到，零改動）；單位上方顯示狀態圖示列。
- **生命週期**：持有者回合開始結算——毒傷（含浮字/log/陣亡 + `checkBattleEnd`）、暈眩者 `exhaust` 跳過；玩家於 `startPlayerTurn`、敵方於 `runEnemyTurn` 各自結算。
- **沉默**：封鎖主動特技按鈕與施放。
- **掛載來源**：`EquipmentDef.onHitStatus`（命中附加，可設 chance），於 `performSwing` 命中時套用。示範：**惡龍弓→毒（2 回合×3）**、**冰霜之書→攻↓（2 回合 -3）**。stun/silence 框架已就緒，留給未來技能/BOSS 來源（掛在武器上每擊觸發會過強）。
- 註：狀態未進戰中存檔（讀檔清空）；`stanceMods`（減傷倍率）暫保留獨立，未併入此框架。

### ✅ G6. 深度機制 #4 — 羈絆 / 支援系統（Wave 6 已完成戰鬥核心）

**戰鬥羈絆**：特定武將對相鄰時，攻擊獲得 `BOND_ATK_MUL`(×1.15) 加成。

- `data/bonds.ts` 定義羈絆對（君臣之誼/並肩之約/神射姊妹/法米爾師徒/天地疾風/不破之盾）+ `findBond`。
- `BattleScene.bondInfo()` 檢查相鄰是否有羈絆夥伴；`performSwing` 與 forecast 套用 `bondMul`，log/預判顯示羈絆名。
- 鼓勵把有故事關聯的角色擺在一起 → 位置戰術 + 情感連結。
- **後續（內容工作，需寫對白）**：每對掛「支援對話」綁進劇情（火紋 C/B/A）—— 屬 cutscenes 內容，非引擎工作。

### ✅ G7. 敵方 AI 升級（Wave 7 已完成第一輪）

在既有「期望傷害評分」上加上 bounded 啟發式：

- **殘血撤退**（normal/hard）：HP < 30% 且無「安全擊殺」（能殺且不被反殺）機會時，改撤退到離玩家最遠的掩體格、不攻擊。
- **側背擊走位**（normal/hard）：選攻擊位置時主動偏好能側/背擊目標的格（接 Wave 4，背 +20 / 側 +8 分）。
- **難度連動智商**：easy 維持原本較魯莽行為（不撤退、不找背擊）；normal/hard 啟用上述。
- 既有的集火（殘血/必殺加成）、風箏（反擊扣分）、掩體偏好、自殺迴避保留。
- **後續**：保護治癒兵/BOSS、角色化行為（攻/守/支援）、hard 專屬威脅地圖（避免被多人圍毆）。

### ✅ G8. 難度與無障礙（Wave 8 已完成 — Part A 戰鬥翻新收尾）

- **難度分層深化**：除既有 `enemyAttackMul`(0.7/1.0/1.3) 與 Wave 7 AI 智商連動，新增 `playerDefenseMul`（easy ×1.2 / normal 1.0 / hard ×0.85）——休閒玩家更耐打、困難更脆。`computeDamage` 在守方=玩家時套用。
- **色盲輔助**（暫停選單可切）：剋制標示加形狀（剋🔺 / 弱▼），不靠顏色辨識；forecast 與 log 都生效。
- **常駐威脅範圍**（暫停選單可切）：開啟後戰鬥啟動即自動開威脅疊圖，每回合自動重繪，無須手動切換。
- 既有的文字放大（HUD 重設計 + Wave 1 浮字加大）、動畫速度、主題切換、可手動切換的威脅疊圖（⚠ 鈕）皆保留。

> 你已選「先免費上架（作品集）」→ 不需 StoreKit、不需帳號系統，打包大幅簡化。

**打包路線（建議：Capacitor）**

| 路線 | 說明 |

| **Capacitor（推薦）** | web→iOS 業界標準，把 `dist/` 包進 WKWebView，文件最完整、社群最大；雖免費版用不到內購，但日後若想加付費功能擴充最平滑。桌面版**繼續用現有 Tauri**。 |

## Part E — 上架素材與合規（免費 App）

1. **App Store Connect**：建立 App 紀錄、截圖（6.7"/6.5" 等）、描述、關鍵字、支援網址。
2. **隱私標籤**：勾「**不收集資料**」（屬實，因零網路）→ 最容易過。但**仍需提供隱私政策 URL**（可放一頁簡單頁面到 GitHub Pages）。
3. **年齡分級**：奇幻暴力、黑暗主題 → 問卷後約落在 9+/12+。
4. **出口合規**：未用非標準加密 → 通常可宣告豁免。
5. **內容授權（最大法律風險，務必先確認）**：
   - **AI 美術**（Nano Banana / Gemini）：確認 Google 條款允許「商業 / 公開發佈」用途。
   - **BGM / SFX**：確認來源授權（若 Suno/Udio 生成，商用權依方案而定；若購買素材，保留授權證明）。
   - App Store 目前不禁 AI 內容，但**你必須擁有發佈權利、不得侵權**。免費發佈同樣需要。
