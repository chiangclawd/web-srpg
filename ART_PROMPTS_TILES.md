# 🎨 戰場地形 Tile — Nano Banana Prompt

> 為戰場 4 種地形產 12 張 painterly anime hex 形圖塊（每種 3 變體散布打散重複感）。
> **程式已就緒**：`assetManifest.ts` / `BootScene.ts` / `BattleScene.drawBoard()` 全 wire。
> Pipeline：把 PNG 丟進 `public/assets/tiles/<key>.png` push 即生效；缺檔仍會回到色塊版。

---

## 📐 共通規格

- **形狀**：**pointy-top hex**（六邊形，尖角朝上下）
- **尺寸**：138 × 160 PNG（2x，遊戲內顯示 69×80）
- **背景**：**純白底**（Nano Banana 無法產透明，產完再自行去底）— hex 形狀的 6 條邊以外的 4 個角必須是純白
- **無文字、無邊框、無浮水印**（程式會在 hex 邊緣自動畫格線）
- **風格**：painterly anime — 賽璐珞著色 + 戲劇性光影 + 飽和度中等，跟既有立繪 / CG 同 family
- **無人物無單位**（純地形材質）

---

## 🎬 共通風格錨定（新對話只貼一次，全 12 張共用）

```
我接下來會請你連續產 JRPG 戰場 hex 地形圖塊。每次只會給你一張的描述，請只產那一張。

風格基準（每張都要符合）：
- 138×160 pointy-top hex 形（尖角朝上下）
- **底色純白**（hex 形狀的 6 條邊以外的 4 個角必須是 100% 純白 #ffffff，方便後續去底）
- painterly anime 風格 — 賽璐珞著色 + 柔和光影
- 飽和度中等、色塊明確、有手繪溫度，**不要 3D 渲染感**
- 沒有人物、沒有文字、沒有邊框（程式會自動畫格線）
- 「俯視角」45 度視角，能看出地形特徵但不是純頂視
- 光源固定左上 45 度

每種地形之後會分 3 個變體（細節略不同但同色系、同風格），讓地圖中相鄰相同地形不會像複製貼上。
請每次只產一張，產完等我下一張的描述。

請確認你理解，回我「了解，每次只產一張」，然後等我給第一張描述。
```

等回「了解，每次只產一張」後才依序貼下方 12 段 prompt（每段獨立一張）。

## 🎨 去底建議工具

| 工具 | 操作 | 適合 |
|---|---|---|
| **Preview**（macOS 內建）| 「智慧 Lasso」→ 選白底 → 刪除 | 1-2 張快速處理 |
| **Photoshop** | 「魔術棒」→ 公差 32 → 選白色 → Delete | 需要精修 hex 邊角 |
| **remove.bg** | 上傳 → 一鍵去底 → 下載 PNG | 12 張批次最快 |
| **GIMP**（免費）| Select by Color → Delete | 全免費替代 |

去底後**務必檢查**：hex 形狀邊緣的 4 個角是透明（棋盤格紋路），主體 hex 內仍完整。

## 🔄 不滿意時的修圖指令

### 純白底沒清乾淨（hex 邊緣有殘色）
```
保持構圖跟畫風，但 hex 形狀以外的 4 個角必須是 100% 純白 #ffffff（方便去底），不要灰邊、不要漸層。重新產。
```

### 變體間色差太大（看起來像不同地形）
```
保持構圖，但讓這張的主色調跟前一張更接近 — 用相同的色塊起手，只在細節（樹的位置、岩石分布）上差異化。
```

### 太陰沉 / 太亮
```
保持構圖跟筆觸，但把整體亮度調暗/提亮 15%，光源方向（左上 45 度）保持不變。
```

### 邊緣銳利不像 painterly
```
保持構圖跟色調，但讓邊緣更柔和、筆觸感更明顯 — 不要太接近 vector / clean line 的風格。
```

### 河流看起來太靜
```
保持構圖，但加強水流動感 — 波紋線條更明顯、白色高光分布更生動，要有「流動」而非「鏡面」感。
```
